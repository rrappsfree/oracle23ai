Frequently Asked Questions
What is Oracle Database 23ai? +
Oracle Database 23ai is Oracle's latest long-term support release, designed from the ground up for AI workloads. It includes native VECTOR data types, AI Functions (DBMS_VECTOR), and deep integration with machine learning pipelines — all within the same trusted relational engine.
Is Oracle 23ai free to use? +
Yes. Oracle offers a free edition of 23ai (Oracle Database Free) with limits of 2 CPUs, 2GB RAM, and 12GB user data. It's free for development and testing without requiring an Oracle license.
How does VECTOR_DISTANCE work? +
VECTOR_DISTANCE is a built-in SQL function that computes similarity between two VECTOR values. It supports cosine, dot product, Euclidean, Manhattan, and Hamming metrics. Use it in ORDER BY to find the nearest neighbors to a query embedding.
Can I migrate from Oracle 19c to 23ai? +
Yes. Oracle supports direct upgrade paths from 19c to 23ai using DBUA, manual upgrade scripts, or Data Pump export/import. Review the Pre-Upgrade Information Tool (preupgrade.jar) output and address all warnings before proceeding.

System Preparation
Update System & Install Pre-requisites

Start with a fully updated system, then install the Oracle pre-install meta-package. This single package configures kernel parameters, system limits, and creates the oracle OS user automatically.

bash 
# Update the system first
sudo dnf update -y
sudo reboot

# Install the Oracle pre-install package
# This configures kernel params, limits, and creates the oracle user
sudo dnf install -y oracle-database-preinstall-23ai
ℹ The oracle-database-preinstall-23ai meta-package is the recommended way to prepare your system. It handles all dependency installation and kernel tuning automatically.
2
Download
Download the RPM Package

Download the Oracle Database 23ai Free RPM. You may need to accept Oracle's license agreement through their web interface before the direct download works.

bash 
# Navigate to /tmp and download the RPM
cd /tmp
wget https://download.oracle.com/otn-pub/otn_software/db-free/oracle-database-free-23ai-1.0-1.el9.x86_64.rpm

# Verify the download (~1.2 GB)
ls -lh oracle-database-free-23ai-*.rpm
⚠ If the direct download fails, visit oracle.com/database/free/get-started and download the RPM manually after accepting the license agreement.
3
Installation
Install the RPM Package

Install the RPM using dnf localinstall. This creates the Oracle software directory, systemd service, and base directory structure.

bash 
sudo dnf localinstall -y oracle-database-free-23ai-*.rpm

The installation creates:

Software/opt/oracle/product/23ai/dbhomeFree
Data files/opt/oracle/oradata/FREE
Logs/opt/oracle/diag/rdbms/free/FREE/trace/
Serviceoracle-free-23ai.service
4
Configuration
Configure the Database

Run the configuration script to create the FREE container database and FREEPDB1 pluggable database. This sets up automatic startup and generates initial passwords.

bash 
sudo /etc/init.d/oracle-free-23ai configure
⚠ Save the passwords displayed during configuration! The script generates the initial SYS, SYSTEM, and PDBADMIN passwords — you will need them to connect. Change them immediately after installation.
5
Post-Installation
Environment Setup & Enable Auto-Start

Configure the Oracle environment variables for the oracle OS user, then enable the systemd service so the database starts automatically on boot.

bash 
# Switch to oracle user and create environment profile
sudo su - oracle
cat > ~/.bash_profile << 'EOF'
export ORACLE_BASE=/opt/oracle
export ORACLE_HOME=/opt/oracle/product/23ai/dbhomeFree
export ORACLE_SID=FREE
export PATH=$ORACLE_HOME/bin:$PATH
export LD_LIBRARY_PATH=$ORACLE_HOME/lib:/lib:/usr/lib
export CLASSPATH=$ORACLE_HOME/jlib:$ORACLE_HOME/rdbms/jlib
EOF
source ~/.bash_profile

# Enable and start the systemd service
sudo systemctl enable oracle-free-23ai
sudo systemctl start oracle-free-23ai
6
Verification
Verify the Installation
bash 
# Check the service status
sudo systemctl status oracle-free-23ai

# Connect as SYSDBA and verify version
sudo su - oracle
sqlplus / as sysdba

SQL> SELECT banner FROM v$version;
SQL> SHOW PDBS;
SQL> EXIT;
✓ You should see FREEPDB1 listed in SHOW PDBS output with READ WRITE status. The database is ready.
7
Security
Change Default Passwords & Create App User

Change the auto-generated passwords immediately. Then create a dedicated developer account instead of using administrative accounts for day-to-day work.

sql 
-- Connect as SYSDBA
ALTER USER SYS IDENTIFIED BY YourNewSysPassword;
ALTER USER SYSTEM IDENTIFIED BY YourNewSystemPassword;

-- Switch to FREEPDB1 and change PDBADMIN password
ALTER SESSION SET CONTAINER = FREEPDB1;
ALTER USER PDBADMIN IDENTIFIED BY YourNewPdbAdminPassword;

-- Create a developer user
CREATE USER appdev IDENTIFIED BY YourAppPassword;
GRANT CONNECT, RESOURCE TO appdev;
GRANT DB_DEVELOPER_ROLE, UNLIMITED TABLESPACE TO appdev;

-- Test developer user connection
-- sqlplus appdev/YourAppPassword@localhost:1521/FREEPDB1

Pre-Installation
Prepare Windows Environment

Before running the installer, prepare the Windows environment to meet Oracle's requirements.

✓ Run as a Windows account with Administrator privileges
✓ Disable UAC: Control Panel → User Accounts → Change UAC Settings → Never Notify
✓ Allow TCP port 1521 through Windows Firewall
✓ Install .NET Framework 4.8 if not already present
✓ Ensure ORACLE_HOME and TNS_ADMIN environment variables are NOT already set
✓ Apply all latest Windows Updates
2
Download
Download Oracle Database 23ai Free

Go to oracle.com/database/technologies/oracle-database-software-downloads.html and download Oracle Database 23ai Free for Windows (64-bit). Extract the .zip to a local folder such as C:\Oracle23ai.

⚠ Ensure the installation directory and all parent directories cannot be modified by the Authenticated Users group — this is a security requirement from Oracle.
3
Installation
Run the Oracle Universal Installer

Open a Command Prompt as Administrator, navigate to the extracted folder, and run setup.exe. Follow the InstallShield wizard.

1 In the Welcome window, click Next
2 Accept the license agreement and click Next
3 The installer runs a prerequisite check — resolve any failures before continuing
4 In Oracle Database Information, set a single password for SYS, SYSTEM, and PDBADMIN — follow Oracle's password guidelines
5 Review the Summary and click Install
6 When complete, the "Oracle Database Installed Successfully" window appears

Quick Start
Run Oracle 23ai in a Container

The fastest way to run Oracle 23ai. Gerald Venzl's community images are widely used for development and support multiple variants (slim, regular, full).

bash 
# Slim image — fastest pull, minimal footprint
docker run -d \
  --name oracle23ai \
  -p 1521:1521 \
  -e ORACLE_PASSWORD=StrongPass1 \
  gvenzl/oracle-free:23-slim

# Monitor startup — wait for "DATABASE IS READY TO USE!"
docker logs -f oracle23ai
✓ When you see "DATABASE IS READY TO USE!" in the logs, the database is fully started and accepting connections on port 1521.
2
Podman (Apple Silicon / ARM)
Run with Podman on ARM / macOS
bash 
# Podman command (same syntax as Docker)
podman run --rm --name oracle23ai \
  -p 1521:1521 \
  -e ORACLE_PASSWORD=StrongPass1 \
  gvenzl/oracle-free:23-slim
3
Persistent Storage
Run with Data Persistence

By default the container is ephemeral — data is lost when the container stops. Mount a volume to persist your data between restarts.

bash 
# Create a named volume for persistence
docker volume create oracle-data

# Run with persistent volume
docker run -d \
  --name oracle23ai \
  -p 1521:1521 \
  -e ORACLE_PASSWORD=StrongPass1 \
  -v oracle-data:/opt/oracle/oradata \
  gvenzl/oracle-free:23-slim
4
Connect & Create User
Connect and Create a Developer User
bash 
# Connect to SQL*Plus inside the container
docker exec -it oracle23ai sqlplus / as sysdba

-- Inside SQL*Plus: switch to FREEPDB1 and create a developer user
ALTER SESSION SET CONTAINER = FREEPDB1;

GRANT DB_DEVELOPER_ROLE, UNLIMITED TABLESPACE
  TO developer IDENTIFIED BY "free";

-- Connect as the new developer user
-- sqlplus developer/free@localhost:1521/FREEPDB1
ℹ The default PDB created in the container is named FREEPDB1. Use SHOW PDBS; inside SQL*Plus to confirm.
